代码规范 ​

HTML ​

  • 属性引号：使用双引号

  • 属性换行：标签属性超过三个的时候需要换行展示

  • 属性名：全小写，用中划线做分隔符

    例如：class="hello-world"

  • 行内样式：禁止在html内写行内样式

  • 标签嵌套：应尽量减少html标签嵌套

  • 使用标签：应正确使用标签，不要全部都用div，还有ul、li、h1、a等

  • doctype: 在页面开头引入 <!DOCTYPE html>

  • lang属性：在html标签添加正确的 lang="en-us"标签

  • 字符编码：添加meta标签，指定对应字符编码。

    例如：<meta charset="UTF-8">

  • IE兼容模式：用meta标签指定什么版本的IE渲染

    <meta http-equiv="X-UA-Compatible" content="IE=Edge">

  • 语义化标签：尽量使用语义化标签

    例如：nav、header等

  • HTML通用模板

    html
    <!DOCTYPE html>
<html lang="zh">
<head>
    <meta charset="UTF-8">
    <title>网页标题</title>
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="renderer" content="webkit">
    <meta name ="viewport" content ="initial-scale=1, maximum-scale=3, minimum-scale=1, user-scalable=no">
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <script src="index.js"></script>
</body>
</html>

CSS ​

  • 选择器：应使用类选择器，避免使用ID选择器

  • 缩进：两个空格缩进

  • 命名：类名使用小写字母，以中划线分隔，id采用驼峰式命名

  • 分号：必须以分号结尾

  • 属性空格：属性名后必须有一个空格，

    color: #000;

  • 选择器空格：选择器和 {之间必须有一个空格，

    .class-name {}

  • 引号：属性值中使用的引号应使用单引号

    例如： font-family: 'open sans', arial, sans-serif;

  • 空行：css规则之间需要空行，不能紧挨着上一个css规则

  • 单位：统一常用单位，uniapp项目中不要px和rpx单位混用。

  • 颜色值：颜色16进制用小写字母，颜色16进制尽量用简写。

    .element {
    color: #abcdef;
    background-color: #012;
}

  • 空规则：不允许有空的规则

  • 空样式：不要有空的style标签，style标签中没有样式时应删除style标签

scss ​

  • 嵌套：可以嵌套的尽量嵌套，不要写在嵌套之外，嵌套层级过深时必须要写注释
  • 主题色：项目定义主题色变量的情况下，应在Css中引用主题色，不要在Css规则中单独写主题色

JavaScript ​

通用规范 ​

缩进 ​

缩进统一使用2个空格

引号 ​

统一使用单引号

分号 ​

行结尾不需要写分号

console ​

非必要不要有console

等号 ​

推荐使用全等===

// 推荐
if (age === 30) {
  // ......
}
// 不推荐
if (age == 30) {
  // ......
}

花括号 ​

使用花括号包裹所有的多行代码块

// 推荐
if (true) {
  // TODO ...
}
// 不推荐
if (true) // TODO ...

注释规范 ​

  • 单行注释：使用//作为单行注释，在注释内容前插入一个空格。

    // 单行注释

  • 多行注释：以/*开头，*/结尾，注释内容前后加一个空格

    /*
 * 第一行注释
 * 第二行注释
 */

  • 方法注释

    /**
 * 方法功能描述
 * @param {*} 参数
 * @example 使用示例(可选)
 * @return 返回值(可选)
 */

Vue ​

ref、reactive ​

正确使用ref和reactive，基础数据类型使用ref，数组和对象的引用类型使用reactive

reactive中使用数组时不要直接将数组赋值给reactive，应该将数组放到对象的属性中

// 不推荐
const data = ref({
  name: 'zhangsan'
})

const boo = reactive(true)

const arr = reactive([])
// 推荐
const boo = ref(true)

const data = reactive({
	name: 'zhangsan'
})

const arr = reactive({
	value: []
})

计算属性 ​

在html标签中判断条件有多个的时候要用computed计算属性

// 不推荐
<template> 
  <div v-if="(detail?.bill_status === '46' || detail.bill_status === '99') && detail?.is_return !== '1'"> 
   ... 
  </div> 
</template>

// 推荐
<template>
  <div v-show="isShow(detail)"> 
   ... 
  </div>
</template>

变量换行 ​

在模板中使用变量时不要换行展示

// 不推荐
<view class="app-font-size-xl app-blod">{{
	orderStatusInfo().text
}}</view>
// 推荐
<view class="app-font-size-xl app-blod">
	{{ orderStatusInfo().text }}
</view>

多属性分行写 ​

超过三个属性要分行写,属性顺序 静态属性 -> 动态属性 -> 方法

html
<template> 
   <scroll 
      ref="scrollWrap" 
      class="home-scroll-warp" 
      :data="homeData" 
      :pullDownRefresh="true"
      @click="pullingUpGetMore" 
   />
</template>

v-if、v-show ​

正确使用v-if、v-show，不要同时使用v-show和v-if

css作用域 ​

页面和组件内的style标签需要写scoped

// 不推荐
<style>
.btn-close {
  background-color: red;
}
</style>
// 推荐
<style scoped>
.btn-close {
  background-color: red;
}
</style>

闭合组件 ​

当组件没有slot内容时使用闭合组件

// 不推荐
<MyComponent></MyComponent>
// 推荐
<MyComponent/>

prop命名 ​

在声明 prop 的时候，其命名应该始终使用 小驼峰，在使用prop的时候应该使用全小写中间横杠的写法

// 不推荐
props: {
  'greeting-text': String
}

<WelcomeMessage greetingText="hi"/>
// 推荐
props: {
  greetingText: String
}

<WelcomeMessage greeting-text="hi"/>
